# Thymian documentation website

[![Built with Starlight](https://astro.badg.es/v2/built-with-starlight/tiny.svg)](https://starlight.astro.build)

```
npm create astro@latest -- --template starlight
```

Added Nx on top of Astro using https://nx.dev/showcase/example-repos/add-astro.

## 🚀 Project Structure

Inside of your Astro + Starlight + Nx project, you'll see the following folders and files:

```
.
├── public/
├── src/
│   ├── assets/
│   ├── content/
│   │   └── docs/
│   └── content.config.ts
├── thymian/
├── astro.config.mjs
├── nx.json
├── package.json
├── project.json
└── tsconfig.json
```

The _thymian_ repository is embedded in the thymian folder as a git submodule. When you run `npm run dev`, the docs are copied from the workspaces in the thymian directory into the `src/content/docs/` directory, replacing any existing files.` Feel free to check out [GitHub's blog entry on git submodules](https://github.blog/open-source/git/working-with-submodules/).

Starlight looks for `.md` or `.mdx` files in the `src/content/docs/` directory. Each file is exposed as a route based on its file name.

## Documentation Generation from Dependencies

The astro-docs project automatically generates reference documentation from dependent packages at build time using Nx's dependency graph:

- **CLI Documentation**: The `generateCliDocs` target runs `generate-docs` on all workspace projects that have this target. These projects use oclif to generate CLI reference documentation directly into `astro-docs/src/content/docs/references/`.

- **Schema Documentation**: The `generateSchemaDocs` target runs `generate-schema-docs` on all workspace projects that have this target. A script processes JSON schema definitions and generates markdown documentation for plugin configuration options.

- **Dependency Detection**: The project uses `implicitDependencies: ["*"]` (with exclusions for non-plugin packages) and Nx's `^target` syntax to automatically detect which projects have changed. This ensures docs are regenerated only when necessary.

- **Build Flow**: `build` → depends on → `increaseCliDocsHeaderLevels` + `generateSchemaDocs` → which depend on → `generateCliDocs` → which runs → `^generate-docs` on all dependencies.

To add documentation generation to a new package, simply add a `generate-docs` and/or `generate-schema-docs` target to its `project.json`. Both targets should include `dependsOn: ["build"]` to ensure the package is built before documentation generation runs.

> [!WARNING]
> The _content/docs/references/\*\*/cli.md_ files include content that is generated by oclif at astro build time. After the generation, the header levels of the whole file are automatically increased by one to match the Starlight documentation structure.
>
> Make sure to decrease your intended header levels by one when you edit these files and never commit the files with already fixed header levels and don't commit them with the generated content! E.g., if you want `## A level 2 heading` write `# A level 2 heading` instead. This is only for the _cli.md_ files under references.

Images can be added to `src/assets/` and embedded in Markdown with a relative link.

Static assets, like favicons, can be placed in the `public/` directory.

## Versions

This project uses [starlight-versions](https://starlight-versions.vercel.app/getting-started/).

Add the starlight-versions plugin as starlight plugin when you create the first version:

```js
// ...
starlight({
  // ...
  plugins: [
    starlightVersions({
      versions: [{ slug: '1.0' }],
    }),
  ],
});
```

There is always the current version and defined archived versions. To create a new version, you add the name (version) for the current version and run `npm run dev`. That will archive the current version under the name. Then you can then go on adjusting the documentation for the new current version. See the above link for details.

## 🧞 Commands

Don't use the astro CLI commands directly! Use the Nx commands instead.

You miss a command? Add it to the `project.json` file.

All commands are run from the root of the project, from a terminal:

| Command                | Action                                       |
| :--------------------- | :------------------------------------------- |
| `npm install`          | Installs dependencies                        |
| `nx run astro:serve`   | Starts local dev server at `localhost:4321`  |
| `nx run astro:build`   | Build your production site to `./dist/`      |
| `nx run astro:preview` | Preview your build locally, before deploying |
