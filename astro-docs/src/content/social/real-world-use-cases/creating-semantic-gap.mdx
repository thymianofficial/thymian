  <PostText platform="reddit" label="Post body">
Last week we explained what the **Semantic Gap** is.

This week: how easy it is to create one.

Now assume you implement a `GET` endpoint that is not safe and not idempotent.

User agents will assume they can safely repeat that `GET` request.

So a crawler, SDK, browser, or AI agent may call it again, retry it, or follow it automatically.

But if your origin server changes state every time it receives that request, you now have a real problem.

The user agent interprets `GET /resource/123` as a safe read.

The server interprets the same request as a state-changing operation.

That difference in interpretation is the Semantic Gap in action.

Docs: https://thymian.dev/introduction/http-conformance/

Have you seen an unsafe `GET` endpoint in the wild?
