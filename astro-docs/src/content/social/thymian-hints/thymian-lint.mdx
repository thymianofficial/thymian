{`\`thymian lint\` checks your OpenAPI specification against HTTP conformance rules — no server required. It catches things like missing \`WWW-Authenticate\` headers on 401 responses or incorrect status codes before you write any implementation code.

The workflow is simple:

1. Point it at your OpenAPI file
2. Review the findings (each one tells you where, why, and which rule)
3. Fix and rerun

It's designed for the design stage, where changes are cheapest. Works great as a CI quality gate too.

Full tutorial: https://thymian.dev/tutorials/thymian-lint-workflow/`}
