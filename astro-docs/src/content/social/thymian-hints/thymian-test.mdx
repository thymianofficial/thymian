{`\`thymian test\` sends real HTTP requests to a running API and validates the responses against HTTP conformance rules.

It catches implementation drift — places where the live behavior no longer matches HTTP expectations or your intended design. Things like:

- Framework defaults that add unexpected headers
- Missing middleware
- Authentication flows that don't follow the spec
- Env-specific behavior from proxies

The workflow:

1. Generate request samples from your OpenAPI spec with \`thymian sampler init\`
2. Point \`thymian test\` at your running server
3. Review findings — each one shows what the server actually returned vs. what the rules expect
4. Fix and rerun

If \`thymian lint\` asks "does the spec look correct?", then \`thymian test\` asks "does the running API actually behave correctly?"

Full tutorial: https://thymian.dev/tutorials/thymian-test-workflow/`}
  </PostText>
  <SocialImage type="code-snippet" platform="reddit" lang="bash" codeLabel="thymian test" promoted code={`npx thymian test --spec openapi:openapi.yaml --target-url http://localhost:3000`} />
