{`\`thymian analyze\` lets you take a HAR file — exported from your browser, an API client, or a proxy — and validate the recorded traffic against HTTP conformance rules.

It stores the traffic in a SQLite database and translates rules into SQL queries, so it handles large datasets efficiently.

Useful when:
- You want to inspect traffic from staging or production
- Proxies, gateways, or caches influence HTTP behavior
- You want to validate what actually happened, not what should happen

You can also pass an OpenAPI spec to check whether the observed traffic matches your API description:

\`\`\`
npx thymian analyze --spec openapi:./openapi.yaml --traffic har:./traffic.har
\`\`\`

Full tutorial: https://thymian.dev/tutorials/thymian-analyze-workflow/`}
